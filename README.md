# Arbonus Portal

Закрытый корпоративный портал **«Бонусы»** на **Next.js + Supabase + Docker + nginx**.

Проект предназначен для внутреннего использования сотрудниками компании и работает по модели:

- закрытый вход по email + пароль
- доступ только для сотрудников из allowlist
- подтверждение почты
- Cloudflare Turnstile на регистрации
- защищенный портал на основе Tilda-экспорта
- защищенные ссылки на ресурсы через `/r/[slug]`
- приватные файлы через Supabase Storage
- деплой на сервер в Docker
- reverse proxy через nginx
- отдельный поддомен `bonus.art-rest.com`

# 1. Что умеет проект

## Авторизация и доступ
- вход по email и паролю
- подтверждение почты
- сброс пароля
- повторная отправка письма подтверждения
- доступ к порталу только после успешной проверки allowlist
- отключение доступа сотруднику после удаления из allowlist

## Безопасность
- серверная регистрация
- Cloudflare Turnstile
- rate limiting на чувствительных маршрутах
- security events в БД
- private Storage buckets
- signed URLs для защищенных файлов
- security headers
- ограничение разрешенных внешних доменов для redirect-ресурсов
- Before User Created Hook в Supabase Auth для блокировки прямого signup в обход allowlist

## Контент портала
- защищенный HTML-портал на основе Tilda-экспорта
- раздача Tilda-ассетов только после проверки доступа
- защищенные ресурсы через `/r/[slug]`
- поддержка многостраничного Tilda-экспорта
- возможность бесшовно обновлять верстку портала из ZIP-архива Tilda

# 2. Бизнес-логика проекта

## Как сотрудник получает доступ
Сотрудник:
1. знает свой внутренний логин `UID`
2. вводит `UID`, email, пароль и проходит капчу
3. система проверяет, что `UID` есть в allowlist
4. система создает регистрацию только если проверка пройдена
5. сотрудник подтверждает email
6. после этого может войти в портал

## Что считается allowlist
Источник allowlist — Excel-файл с колонками:

1. `UID`
2. `First name`
3. `Middle name first letter`
4. `Last name`
5. `Store`

`UID` — это логин сотрудника, например `abc1234`.

# 3. Текущая архитектура

## Приложение
- **Next.js 16**
- **App Router**
- **Supabase Auth**
- **Supabase Postgres**
- **Supabase Storage**
- **Docker**
- **nginx**
- **Cloudflare Turnstile**

## Текущая серверная схема
- контейнер приложения бонусов слушает **8081**
- nginx на сервере слушает **80/443**
- nginx проксирует `bonus.art-rest.com` на `127.0.0.1:8081`

## Соседний сайт
На этом же сервере уже работает отдельный сайт:

- `https://podrabotka.art-rest.com/`

Он должен жить независимо и не ломаться при изменениях бонусов.
